# Architecture — GymChuot Prototype

## Key Files

### `src/App.jsx` (~500 lines)
The entire UI prototype lives here. Two main panels:
- **Left panel**: Workout Builder — exercise library (search + filter) + drag-and-drop canvas
- **Right panel**: Messenger — student list + chat bubbles + workout card renderer

State is all local React state (no router, no global store). This is intentional for the prototype phase — production will split into proper pages with Supabase state.

### `src/useExercises.js`
React hook that loads and searches `data/exercises.json`. Exposes:
- `exercises` — full list
- `filteredExercises` — filtered by search query + category
- `setQuery`, `setCategory` — filter setters

### `data/exercises.json`
Generated by the data pipeline scripts. 800+ exercises from free-exercise-db, extended with GymChuot fields (`nameVi`, `commonMistakes`, `videoUrl`, `gymchuotVerified`, `tags`).

## Data Pipeline

```
scripts/import-exercises.js   # Pulls from free-exercise-db, transforms to GymChuot schema
scripts/translate-vi.js       # Translates exercise names to Vietnamese via Anthropic API
```

Run: `npm run data:import` then `ANTHROPIC_API_KEY=sk-... npm run data:translate`

## Drag & Drop
Currently uses HTML5 native DnD API. Production will migrate to **dnd-kit** for touch support and smooth animations.

## Production Stack (planned)
Next.js 14 + dnd-kit + Tailwind + Framer Motion / Supabase (Auth + DB + Realtime + Storage) / Vercel
