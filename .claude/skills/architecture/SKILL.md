## State Distribution

### App.jsx (cross-cutting)
- `activeTab` вЂ” drives major view
- `exercises`, `workoutName` вЂ” workout being built
- `selectedStudent` вЂ” used by builder + messenger
- `messages` вЂ” shared between sendWorkout and chat display
- `workoutSent`, `showSentToast` вЂ” driven by sendWorkout()
- `draggingFromLibrary` вЂ” shared between ExerciseLibrary (sets it) and WorkoutBuilder (reads it)

### Component-local state
- `searchQuery`, `activeCategory` в†’ ExerciseLibrary
- `dragIndex`, `dragOverIndex`, `isDropZoneActive` в†’ WorkoutBuilder
- `chatInput`, `messagesEndRef` в†’ ChatArea
- `expanded` в†’ ExerciseCard, WorkoutMessageCard

## Key Files

### `src/constants.js`
Static prototype data: `EXERCISE_LIBRARY` (grouped by category), `EQUIPMENT_COLOR`, `SAMPLE_STUDENTS`, `SAMPLE_MESSAGES`.

### `src/useExercises.js`
Hook that loads `data/exercises.json` (800+ exercises from free-exercise-db) with Vietnamese translations. Exports: `useExercises()`, `useExerciseSearch()`, `groupByCategory()`. Not yet wired to the UI вЂ” integration happens when Supabase replaces local state.

### `data/exercises.json`
Generated by the data pipeline. Schema: `{ id, name, nameVi, category, equipment, level, mechanic, force, primaryMuscles, secondaryMuscles, instructions, images, commonMistakes, videoUrl, gymchuotVerified }`. Do not edit by hand.

## Data Pipeline

```
scripts/import-exercises.js   # Pulls from free-exercise-db, transforms to GymChuot schema
scripts/translate-vi.js       # Translates exercise names to Vietnamese via Anthropic API
```

Run: `npm run data:import` then `ANTHROPIC_API_KEY=sk-... npm run data:translate`

## Drag & Drop
Currently uses HTML5 native DnD API. Production will migrate to **dnd-kit** for touch support and smooth animations.

## State Management Approach
Plain prop drilling вЂ” no Context API or Zustand. Depth is max 2вЂ“3 levels. Context/Zustand introduced when Supabase replaces local state (items 001/002).
